# REST API란

## REST란?

**Re**presentational **S**tate **T**ransfer

네트워크 아키텍쳐 원리모음

**REST 아키텍쳐의 6가지 제한 조건**

* 클라이언트/서버 구조
* 무상태 (Stateless)
* 캐시 처리 가능 (Cacheable)
* 계층화 (Layered System)
* Code on demand (optional)
* 인터페이스 일관성

HTTP에는 이미 위의 REST 원칙이 녹아 들어 있다.

따라서 우리는 Route에만 신경 쓰면 REST하게 사이트를 만들 수 있다.

**REST하게 만든다.** => URL 주소만 보고도 수행하려는 동작을 눈치챌 수 있게 만드는 것
